Pensando como um Programador

Um grande objetivo deste Nanodegree é dar-lhe a prática de pensar como um programador de computador. Programadores de longa data freqüentemente falam sobre como o processo de pensamento de programação os ajudou em outras áreas de sua vida, mas o que significa "pensar como um programador?" Neste Nanodegree vamos nos concentrar em cinco maneiras específicas de pensar que todos os programadores têm que usar o tempo todo. Voltaremos a essas idéias repetidamente em todo o Nanodegree.

  1. Pensamento Procedimental - Criar instruções claras, não ambíguas, para o computador seguir.
  2. Pensamento Abstrato - Encontrar similaridade entre coisas aparentemente diferentes.
  3. Pensamento Sistêmico - Dividir grandes problemas em pequenos pedaços.
  4. Empatia Tecnológica - Entender o que é um computador, como ele funciona e o que ele faz de melhor e pior.
  5. Depuração - Indentificar, insansavelmente a razão pela qual um programa de computador não funciona.

Estrutura básica do HTML - HyperText Markyp Language

HTML significa Hypertext Markup Language. Os documentos em HTML forma a maior partte da estrutura de conteúdo da Web.

O HTML é composto por:

  1. Conteúdo do texto (o que você vê);
  2. Marcação (o que de fato é exibido);
  3. Referências a outros documentos (imagens ou vídeos);
  4. Links para outras páginas.

Exemplos de marcação em HTML:

  • <tag>Escrito entre tags</tag>
  • <b> Essa marcação deixa o texto em negrito </b>
  • <em> Marcação responsável por deixar o texto em itálico </em>
  • <a href="url"> Adiciona um link para algum endereço Web >/a>
  • <img src="url" alt="image">Adiciona um link para imagem externa
  • <br> Pula uma linha (inline) </br>
  • <p> Cria um parágramo (in block) <p>
  • <span> Agrupa alguns elementos em linha </span>
  • <div> Agrupa elementos em block para formatação </div>
